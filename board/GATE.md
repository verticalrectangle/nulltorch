# NullTorch-Board — mechanical gate

Every item below must pass **before human judging** (RUBRIC.md). Any
failing blocking item disqualifies the submission from judging; it is
recorded as "gate-failed".

v1 is a **manual checklist** executed by an operator. Each item notes how
it will be automated later — the target is a headless-browser harness
(Playwright/Chromium driving the bundle from `file://` with network access
disabled) plus a static scanner, so the whole gate becomes one command.

Datasets used throughout: `sample_data/empty.json`,
`sample_data/one_model.json`, `sample_data/many_models.json`.

## G0. Zero external dependencies (BLOCKING — static scan, run first)

- [ ] Scan every file in the submission for external references. **Any hit
      is an automatic gate failure**, enforced before any human judging:
  - `http://` or `https://` URLs in `src`, `href`, `srcset`, CSS `url()`,
    `@import`, or JS string literals that are fetched
  - protocol-relative references (`//cdn...`, `src="//..."`,
    `href="//..."`)
  - `@import url(...)` pulling anything not in the bundle
  - web-font, external-image, analytics, or iframe references
- [ ] No frameworks or libraries: no bundled React/Vue/Svelte,
      D3/Chart.js/Plotly, Tailwind/Bootstrap or any third-party code —
      vanilla HTML/CSS/JS only. (Vendored library source counts as a
      library. Spot-check large JS files for license headers / minified
      vendor signatures.)
- [ ] No build step required: the submission runs exactly as submitted
      (no `package.json` build, no bundler artifacts requiring
      regeneration).
- [ ] All assets are inline or sibling files referenced by relative path;
      the only runtime data input is the local `results.json`.

*Automation later:* static scanner (regex + HTML/CSS parse) over the
bundle; plus a headless run with a request interceptor that fails the gate
if **any** network request is attempted. Exceptions list: none.

## G1. Renders offline with no console errors

- [ ] Open the entry HTML via `file://` with networking disabled
      (airplane mode or a blocked network namespace).
- [ ] Load each of the three datasets; the app renders its primary view.
- [ ] Browser console shows zero errors (uncaught exceptions, failed
      resource loads, CSP violations). Warnings are noted but don't fail.

*Automation later:* Playwright with `context.setOffline(true)`, collect
`console` and `pageerror` events across a scripted tour of all views ×
all three datasets; assert the error list is empty.

## G2. Handles all three sample datasets, including degenerate

- [ ] `empty.json`: meaningful empty state — no crash, no blank page, no
      NaN/undefined/Infinity rendered anywhere.
- [ ] `one_model.json`: all five required views render; side-by-side
      comparison degrades sanely with only one model (no crash, honest
      messaging).
- [ ] `many_models.json`: leaderboard remains usable (no unbounded page
      growth, interactions don't visibly lag). Note: real data may reach
      ~50 models; if a 50-model dataset is provided in a later release,
      it is run under this item.

*Automation later:* scripted dataset-load helper; DOM assertions per view
per dataset; a `50_models.json` generated by `gen_samples.py` gets added
to the matrix.

## G3. DOM assertions — displayed numbers match the JSON (known dataset)

Using `one_model.json` (model `nova-2-pro`), verify displayed values
against the file. Note: this known dataset now carries the **RVC
engine-fidelity column** on its non-delta runs (present on `open_book` and
`closed_book`, absent on `delta`) in addition to the T1–T6 tiers. At
minimum:

- [ ] Leaderboard: the T1 pass count/rate shown for one specific
      (language, condition) row equals `tiers.T1.pass` / `tiers.T1.total`
      from the matching run object.
- [ ] Leaderboard RVC column: on an `open_book` row, the displayed RVC
      value equals `tiers.RVC.pass` / `tiers.RVC.total`; the RVC column is
      visually distinguished from the T1–T6 tiers.
- [ ] Model detail heatmap: one specific tier × language cell equals the
      corresponding run's `pass/total`, including the RVC row/column on a
      non-delta cell.
- [ ] RVC absence on delta: for the `delta` run (which has no `tiers.RVC`),
      the RVC slot renders as empty / "n/a" — not zero, not `NaN`,
      no console error. Confirm nothing in the UI derives a phantom RVC
      value for delta.
- [ ] Memorization gap: the displayed gap for one (language) equals the
      recomputed stock-minus-delta value from the JSON over the **T1–T6
      reading tiers only** (it is derived — the file stores no gap field —
      and RVC is excluded from it).
- [ ] Reproducibility strip: `harness_version`, `fixture_set_hash`, and at
      least one container digest are displayed verbatim (full hash
      available, e.g. via tooltip/copy — a visual ellipsis is fine).
- [ ] Fixture drill-down: one failing category's pass/total matches
      `tiers.<T>.categories.<name>` (verify one T1–T6 category and one RVC
      category, e.g. `tiers.RVC.categories.config_extraction`).

*Automation later:* Playwright locators keyed on stable roles/labels; the
expected values computed from the JSON in the test itself, so the
assertions survive dataset regeneration.

## G4. Keyboard navigability

- [ ] Every interactive control (sort headers, filters, model selectors,
      compare picker, theme toggle, drill-down links) is reachable with
      Tab/Shift-Tab in a sensible order.
- [ ] Focus is visible at every stop.
- [ ] Controls operate via keyboard (Enter/Space to activate, arrows where
      idiomatic); sorting the leaderboard and answering "which model is
      best at T3-Rust?" is possible without touching the mouse.
- [ ] No keyboard traps; overlays/dialogs (if any) close with Esc and
      return focus.

*Automation later:* Playwright keyboard walk — Tab through the page,
assert the focus path covers all controls, drive one full
sort-filter-select flow with key events only.

## G5. No horizontal page scroll at common widths

- [ ] At viewport widths **375 px**, **768 px**, and **1440 px** (any
      common height), for each dataset:
      `document.documentElement.scrollWidth <= document.documentElement.clientWidth`.
- [ ] Wide content (tables, heatmaps, charts) scrolls inside its own
      `overflow-x` container, not the page.

*Automation later:* Playwright `setViewportSize` loop × datasets × views;
evaluate the scrollWidth assertion; screenshot on failure for the record.
