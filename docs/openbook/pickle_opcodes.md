# Pickle opcode reference

Pickle is a **stack machine**. A pickle stream is a flat sequence of one-byte
opcodes, each optionally followed by an argument encoded immediately after the
opcode byte (the *Arg* column below gives the argument's wire encoding).
Executing the stream pushes and pops values on a data stack; the special
`markobject` is a sentinel pushed by `MARK` so that variadic opcodes
(`TUPLE`, `LIST`, `DICT`, `APPENDS`, `SETITEMS`, ...) can pop "everything back
to the most recent mark". A stream is terminated by `STOP`, which pops the
final value: that value is the unpickled object.

**Memo.** The *memo* is a side table mapping indices to already-constructed
objects. `PUT`/`BINPUT`/`LONG_BINPUT`/`MEMOIZE` store the object on top of the
stack into the memo; `GET`/`BINGET`/`LONG_BINGET` push a memoized object back
onto the stack. This is how shared references and self-referential structures
are encoded ‚Äî a correct reader **must** implement the memo (protocol 4 streams
use `MEMOIZE`, which appends to the memo at the next unused index, after
almost every constructed object).

**Protocol markers.** Protocol ‚â• 2 streams begin with `PROTO` (`0x80`)
followed by a one-byte protocol number. Protocol ‚â• 4 streams additionally wrap
content in `FRAME` (`0x95`) records ‚Äî an 8-byte little-endian length followed
by that many bytes of opcodes; framing is a transport-layer hint and a reader
may simply skip over `FRAME` opcodes and their length argument. The *Proto*
column below is the protocol version in which each opcode was introduced;
a reader targeting protocols 2 and 4 must handle every opcode with
Proto ‚â§ 4, and must still accept protocol 0/1 opcodes (they remain legal in
higher-protocol streams).

**Common integer encodings** used by arguments: `uint1`/`uint2`/`uint4`/
`uint8` are unsigned little-endian; `int4` is signed little-endian;
`decimalnl_short`/`decimalnl_long` are ASCII decimal followed by `\n`.
Strings are either newline-terminated (protocol 0) or length-prefixed
(`BINUNICODE` = 4-byte length + UTF-8, `SHORT_BINUNICODE` = 1-byte length,
`BINUNICODE8` = 8-byte length; `BINBYTES`/`SHORT_BINBYTES`/`BINBYTES8`
likewise for raw bytes).
